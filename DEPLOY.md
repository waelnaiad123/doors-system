# دليل التشغيل - المرحلة الأولى (المشاريع والأبواب)

## 1) إنشاء مشروع Supabase (قاعدة البيانات + تسجيل الدخول) - مجاني
1. اذهب إلى https://supabase.com وسجّل حساب مجاني، ثم "New Project".
2. بعد إنشاء المشروع، من القائمة الجانبية: **SQL Editor > New query**.
3. الصق محتوى ملف `01_schema.sql` (المُرسل سابقًا) بالكامل واضغط **Run**.
   هذا يبني كل الجداول والصلاحيات والتقارير دفعة واحدة.

## 2) إنشاء أول مستخدم أدمن (يدويًا - مرة واحدة فقط)
النظام لا يسمح لأي شخص بإنشاء "أدمن" من نفسه، لذلك أول أدمن يُنشأ يدويًا:

1. من Supabase: **Authentication > Users > Add user** → أدخل بريده الإلكتروني وكلمة سر مبدئية.
2. انسخ الـ `User UID` الذي ظهر له.
3. من **SQL Editor** نفّذ (استبدل القيم بين علامتي `<>`):

```sql
insert into public.profiles (id, full_name, role)
values ('<USER-UID-هنا>', 'اسم الأدمن', 'admin');
```

من الآن، هذا الحساب يدخل النظام كأدمن.

> لإضافة مستخدمين تجريبيين آخرين (فني، مدخل بيانات...) بنفس الطريقة مؤقتًا، كرر نفس الخطوتين
> وغيّر `role` إلى: `data_entry` / `technician` / `supervisor` / `engineer` / `delivery_entry`.
> **ملاحظة:** شاشة إدارة المستخدمين (لإنشاء الـ 100 حساب وربطهم بالمشاريع من داخل الموقع مباشرة،
> بدون الدخول لـ Supabase) هي الخطوة القادمة في البناء.

## 3) ربط الموقع بمشروعك
1. من Supabase: **Project Settings > API** — انسخ `Project URL` و `anon public key`.
2. داخل مجلد المشروع، انسخ `.env.example` باسم `.env` والصق فيه القيمتين.

## 4) تجربة محلية (اختياري، يحتاج Node.js مثبت على جهازك)
```bash
npm install
npm run dev
```
يفتح على `http://localhost:5173`

## 5) النشر المجاني على Vercel
1. ارفع مجلد المشروع إلى GitHub (repo جديد).
2. من https://vercel.com سجّل دخول بحساب GitHub → **Add New Project** → اختر الـ repo.
3. في خطوة الإعدادات، أضف Environment Variables:
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`
   (نفس القيم من ملف `.env`)
4. اضغط **Deploy**. بعد دقيقة تقريبًا يديك رابط ثابت مجاني مثل: `your-app.vercel.app`

## ما يعمل الآن
- تسجيل الدخول بالبريد وكلمة السر (لكل دور واجهة مختلفة تلقائيًا).
- شاشة "المشاريع": بحث، إنشاء مشروع، فتح مشروع.
- داخل المشروع: إضافة أبواب يدويًا (مع بنودها)، أو استيراد ملف Excel دفعة واحدة، مع تحميل نموذج جاهز.

## القادم في المرحلة التالية
تسجيل التركيب اليومي للفني (مع التظليل)، اعتماد المشرف، التسليمات، التقارير والنسب المئوية،
وشاشة إدارة المستخدمين والتخصيصات من داخل الموقع.
